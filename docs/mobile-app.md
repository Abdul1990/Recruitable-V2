# Mobile App — Developer Guide

Flutter app for iOS, Android, and Web. Located in `mobile/`.

---

## Prerequisites

- Flutter SDK 3.3+  ([flutter.dev/docs/get-started/install](https://docs.flutter.dev/get-started/install))
- Dart 3.3+
- Xcode 15+ (iOS builds)
- Android Studio (Android builds)
- Firebase project configured ([console.firebase.google.com](https://console.firebase.google.com))

---

## First-time setup

```bash
cd mobile

# Install dependencies
flutter pub get

# Configure Firebase (generates lib/firebase_options.dart)
dart pub global activate flutterfire_cli
flutterfire configure

# Run on connected device or simulator
flutter run
```

## Routing

All routes are defined in `main.dart` using `go_router`:

| Path | Screen |
|---|---|
| `/` | `SplashScreen` — landing + LinkedIn CTA |
| `/auth/linkedin` | `LinkedInAuthScreen` — WebView OAuth |
| `/onboarding/skills` | `SkillSelectionScreen` — pick top 5 skills |
| `/onboarding/location` | `LocationPickerScreen` — APAC country picker |
| `/feed` | `SwipeFeedScreen` — job swipe feed |
| `/matches` | `MatchesScreen` — list of all matches |
| `/matches/:matchId/chat` | `ChatScreen` — message thread |
| `/profile` | `ProfileScreen` — view/edit profile |

Unauthenticated users are redirected to `/` by the GoRouter `redirect` guard.

## Skill → Specialism Mapping

The `resolveSpecialism(List<String> selectedSkills)` function in `data/specialisms.dart` maps the user's 5 selected skills to the best-matching specialism by counting skill overlaps.

```dart
// Example
final skills = ['React', 'Next.js', 'TypeScript', 'NodeJS', 'Postgres'];
final specialism = resolveSpecialism(skills);
// → Specialism(id: 'fullstack', title: 'Full Stack Development', ...)
```

Call `resolveTopSpecialisms(skills, count: 3)` to get the top 3 matches (used for "You could also match as..." suggestions).

## APAC Regions

Defined in `app_config.dart`:

```dart
AppConfig.apacRegions  // List<ApacRegion> — all 6 markets
AppConfig.apacRegionCodes  // ['MY', 'KL', 'SG', 'AU', 'VN', 'ID', 'TH']
AppConfig.isApac('MY')     // true
```

The swipe feed uses `getJobsForUser(userRegionCode: 'MY', ...)` which:
1. Fetches APAC jobs first, sorted with the user's own country at the top
2. Appends non-APAC jobs (US, London) at the end

## Match Scoring

Match scores are computed server-side by calling `/api/match-score`. The Flutter app never calls the Claude API directly.

```dart
final result = await MatchingService.instance.scoreCandidate(
  candidate: profile,
  job: job,
);
// result.score    → 0–100
// result.reason   → "Strong React/TypeScript overlap; candidate lacks Go..."
// result.isMatch  → true if score >= 60 (AppConfig.minMatchScore)
// result.isStrongMatch → true if score >= 85 (AppConfig.topMatchScore)
```

The feed pre-scores all jobs in batches of 5, filters out scores below the threshold, and sorts by score descending before presenting cards to the user.

## CV Upload Flow

```dart
// 1. Pick file
final file = await CvService.instance.pickCv();

// 2. Upload to backend (renders PDF→PNG, strips PII, stores in Firebase Storage)
final url = await CvService.instance.uploadCvAndGetImageUrl(uid: uid, file: file);

// 3. Save URL to user profile
await FirestoreService.instance.saveCvImageUrl(uid, url);
```

The backend (`/api/cv-upload`) does the heavy lifting:
- Renders PDF page 1 to PNG at 2× resolution
- Scans the text layer for emails and phone numbers
- Overlays white redaction boxes over detected PII
- Uploads the clean PNG to Firebase Storage
- Returns the public URL

## Real-time Chat

Messages use Firestore real-time streams. The chat screen subscribes automatically:

```dart
StreamBuilder<List<Message>>(
  stream: FirestoreService.instance.watchMessages(matchId),
  builder: (context, snapshot) { ... },
)
```

Sending a message:
1. Writes to `/messages/{matchId}/thread/{newDocId}`
2. Updates `/matches/{matchId}` → `hasUnreadMessages: true`
3. Firestore security rules verify `senderId == auth.uid`

## Build for Production

### iOS

```bash
flutter build ios --release
# Open mobile/ios/Runner.xcworkspace in Xcode
# Product → Archive → Distribute App → App Store Connect
```

### Android

```bash
flutter build appbundle --release
# Upload mobile/build/app/outputs/bundle/release/app-release.aab
# to Google Play Console → Production
```

### Web

```bash
flutter build web --release
# Output in mobile/build/web/
# Deploy to Vercel or Firebase Hosting
```
